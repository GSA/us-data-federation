Bringing together the United States of data.

The first data ingestion tool built by government, for government. The U.S. Data Federation Tool supports interoperability and standardization across Federal, state and local government. Our playbook highlights common data formats, API specfications, validation methods and metadata vocabularies. Better data, with less hassle.

What is federated data?

Federated data refers to data aggregated from many entities across organizational boundaries (local, state, federal).

What is a federated data effort?

A federated data effort is an attempt to streamline and improve the collection of these data. This might mean agreeing on a standard way to report data across organizations, or it might mean designing tools that make reading in and validating data in many different formats easier.

Why does this matter?

Long term, successful federated data efforts are unlikely if the process by which data are collected, aggregated and validated cannot be improved. Gathering these data in a timely fashion by using tools that are complementary to existing workflows (e.g. they do not add additional time and effort on the data provider) and that are easy to use will help modernize the way the federal government functions and interacts with other government agencies.
